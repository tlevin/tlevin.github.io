Building a simple Chrome Extension

11/13/2015

I was recently able to make a chrome extension as a personal project and I was surprised at the ease of making something work. I wanted to share what I found to be true and where to start.

Easymode: Yo

If you are unfamiliar with a Yeoman generator, it is a tool to build out a basic scaffold for a project. It allows a developer to bypass the tedious chores of making folder structures, the ability to inject dependencies and even to have automated testing out of the box.

I found a great generator for chrome extensions called simply generator-chrome-extension. It installs dependencies for the program with a simple npm install -g generator-chrome-extension.

Next make and enter a folder for your project and run yo chrome-extension and watch the magic. It will walk you through general setup and then give you a simple scaffold to build your extension with.

What is required for a Chrome Extension?

So you want to do it the hard way?

Good! Let's look into what is needed for a basic chrome extension to work.

All you need for a chrome extension is a manifest.json file where you define description, icons, permissions as well as what type of Chrome Extension you are building. I chose to use a popup and there is good documentation here on how to build your manifest.json file.

After you declare your extension and permissions, you just need to develop the extension. Guess what? A Chrome extension popup is simply a html page so you can use any front-end technology to build a responsive, feature rich document. I personally used angular for my front-end on this project and it came out pretty well.

If all goes well you can load your extension into chrome as described in the getting started docs.
