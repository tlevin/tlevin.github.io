Learning subclasses with HR-rogger

10/15/2015

I had the pleasure of diving deep into subclass pseudoclassical instantiation patterns with a project to build a simple interactive stage to move objects around on.

So, what did I do? I turned it into a Frogger game. I created a base class called Obstacle and had Vehicles, Frogger, and RockObstacle as subclasses which inherited some basic methods and properties.

The basic formula for creating a subclass in the pseudoclassical pattern is to:

  1. Call the parent class, set the this context and pass in any parameters that are taken into the subclass that are referenced by inherited properties.

  2. Create a delegation from the subclass prototype to the parent class prototype using Object.create

  3. Point the subclass prototype constructor back to itself to maintain a relationship between instances and the subclass

This is how it looks in code: 

var Frogger = function(top, left) {
  Obstacle.call(this, top, left);
  this.$obstacleNode.addClass('frogger');
  this.left = left;
  this.top = top;
  this.isAlive = true;
  this.hasWon = false;
  this.canStart = false;
};

Frogger.prototype = Object.create(Obstacle.prototype);
Frogger.prototype.constructor = Frogger;

Subclassing makes it easier to create distinct classes that share similar properties and methods. It helped me abstract what the relationship between components in my project. It was my first foray into application design where I had to plan out inheritance prior to starting to actually code.

This project introduced me to using audio in HTML5, CSS animations, and the complexities of building applications from idea to implementation. The challenge to make a viable product in only two days pushed me to learn jQuery, implement object collision and deal with the challenges of CSS.
