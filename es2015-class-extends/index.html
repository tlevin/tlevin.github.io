ES2015 Class extends

12/08/2015

Fun with Fat arrows

How I learned to abandon the ;

I have started writing and thinking of my apps in the new ES2015 standard and wanted to muse on the construction of a subclass using the class SubClassName extends SuperClass { } structure.

Subclassing is standardized

Subclassing in ES2015 is great, it is simple and makes sense. All you need are the constructor method and any methods you add beyond that. Let's run through an example

class Transaction extends db.Model {
  /*Using the class syntax you declare which subclass and superclass*/

  constructor(attributes){
    /*Here you will pass in any params that would be set at instantiation.*/ 

    super()
    /*Make sure you call super and include any parameters you want to be included in the super class*/

    this.tableName = 'transactions'
    this.attributes = attributes
  }

categories(params){
  /*This is the format for methods you want to add to the subclass prototype*/

  return this.hasMany(Category, 'id_Category')
}

My general consensus on the syntax for extending a class in EC2015 is that it is a simple and straightforward abstraction on the subclass instantiation patterns from ES5 and makes a lot of sense if you familiar with them.

Some gotchas I encountered in the use was not calling super in the constructor. It is required in order to form that association. Also the constructor function is required even if you are only adding methods to the subclass and not properties.
